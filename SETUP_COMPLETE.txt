🔐 VALIDATION RULES:

Input Validation:
  ✓ Format: CSV dengan comma separator
  ✓ Header: Exact match required
  ✓ Columns: Exactly 3 (no more, no less)
  ✓ Data Types: Transaction_ID=text, Times=number
  ✓ Range: Times must be > 0
  ✓ Minimum: At least 1 data row

Error Messages:
  "Minimal 2 baris diperlukan"         → Tambah data
  "Kolom harus: ..."                   → Fix header
  "Baris X: Harus ada 3 kolom"         → Fix row
  "... harus angka"                    → Use numbers
  "Waktu tidak boleh negatif"          → Use positive values

════════════════════════════════════════════════════════════════════════════

💡 TIPS & TRICKS:

Data Sizing:
  - Low traffic: Interarrival 4-6 detik
  - Normal traffic: Interarrival 2-3 detik
  - High traffic: Interarrival 1-1.5 detik
  - Test scenarios: Mix of different values

Session Management:
  - Data stored during session (browser active)
  - Lost on refresh/tab close
  - Copy data if need to save elsewhere

Best Practices:
  - Test with 5-10 transaksi dulu
  - Use realistic values
  - Verify KPI cards sebelum simulasi
  - Check histogram distribusi

  1. Session-based storage (lost on refresh)
